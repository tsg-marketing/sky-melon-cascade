# Письмо команде разработки: подключение SEO-пререндера

Привет!

Прошу внести небольшое изменение в процесс деплоя сайта meatmassagers.ru.
Ниже — что это, зачем и что именно нужно сделать.

---

## В чём проблема

Сайт сделан как SPA (React + Vite). При сборке получается **один** файл
`index.html` с одинаковыми `<title>` и `description` — а нужное содержимое
подставляется уже в браузере через JavaScript.

Проблема в том, что внешние SEO-сканеры (Topvisor, Screaming Frog, Serpstat и др.)
и часть поисковых роботов **не выполняют JavaScript**. Они читают сырой HTML с
сервера и поэтому видят на всех страницах **одинаковые title и description**.
Из-за этого сервисы показывают «дубли метатегов», и это вредит SEO.

## Как решаем

В проект добавлен скрипт `scripts/prerender.mjs`. Он запускается **после сборки**
и создаёт для каждого адреса сайта отдельный `index.html` с уникальными
`title`, `description`, `canonical` и og-тегами (соцсети). Данные берутся из того
же источника, что и сам сайт, поэтому теги полностью совпадают с контентом.

Итог: пользователи получают привычное SPA, а сканеры и роботы — готовый HTML с
правильными уникальными метатегами на каждой странице (~360+ страниц).

## Что нужно сделать (2 пункта)

**1. Добавить один шаг в деплой — запуск пререндера после сборки:**
```bash
npm ci
npm run build
node scripts/prerender.mjs      # ← добавить эту строку
```
Требуется Node.js 18+ (уже есть) и доступ в интернет во время сборки к
бэкенд-фиду `functions.poehali.dev` (из него берётся список товаров).

**2. Проверить настройку веб-сервера** — чтобы для вложенных адресов отдавался
сгенерированный `index.html`, а не корневой. Это ключевой момент: без него
сканеры продолжат видеть дубли. Готовые примеры для Nginx и Apache — в файле
`PRERENDER.md` (раздел «Требования к раздаче статики»).

## Как проверить, что заработало

После деплоя (эмулируем сканер — без JS):
```bash
curl -s https://meatmassagers.ru/blokorezki                         | grep -o '<title>[^<]*</title>'
curl -s https://meatmassagers.ru/volchki-myasorubki                 | grep -o '<title>[^<]*</title>'
curl -s https://meatmassagers.ru/blokorezki/blokorezka-imb-2300-638 | grep -o '<title>[^<]*</title>'
```
Заголовки должны быть **разными** на каждом URL. После этого можно перезапустить
проверку в Topvisor — дубли уйдут.

## Файлы в репозитории

- `scripts/prerender.mjs` — сам скрипт пререндера.
- `PRERENDER.md` — подробная техническая инструкция (окружение, настройка
  сервера, обслуживание).
- `PISMO_KOMANDE.md` — этот файл.

Если сборка идёт в закрытом контуре без доступа в интернет — напишите, подготовим
вариант со статическим списком страниц.
